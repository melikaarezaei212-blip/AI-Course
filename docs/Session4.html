جلسه ۴: یادگیری ماشین (بنیادها و تکنیک‌های نظارتی)

تحلیل عمیق مدل‌های رگرسیون، درخت‌های تصمیم، ساختار داده و معیارهای ارزیابی عملکرد

بررسی جزئیات گرادیان کاهشی، تابع احتمال لگاریتمی (Log-Likelihood)، شبه‌کدها و معیارهای Precision، Recall و F1-Score.

بخش اول: طبقه‌بندی بنیادین مدل‌های یادگیری ماشین و نقش داده‌ها

تحلیل جامع پارادایم‌های یادگیری ماشین (Supervised, Unsupervised, Reinforcement) و متدولوژی‌های تقسیم و اعتبارسنجی داده‌ها.

۱.۱. پارادایم‌های اصلی یادگیری ماشین (Learning Paradigms)

یادگیری نظارتی (Supervised Learning)

مدل از روی جفت‌های برچسب‌دار \(\mathcal{D} = \{(\mathbf{x}^{(i)}, y^{(i)})\}_{i=1}^m\) آموزش می‌بیند. هدف، کمینه‌سازی خطای مورد انتظار بر روی توزیع داده‌های واقعی (\(\mathcal{P}\)): \[ \text{Minimize } E_{\mathbf{x}, y \sim \mathcal{P}} [L(y, h(\mathbf{x}))] \]

یادگیری بدون نظارت (Unsupervised Learning)

مدل با داده‌های بدون برچسب (\(\mathcal{D} = \{\mathbf{x}^{(i)}\}_{i=1}^m\)) مواجه است. هدف، کشف ساختارها یا توزیع احتمالی پنهان داده‌ها (مانند خوشه‌بندی یا کاهش ابعاد).

یادگیری تقویتی (Reinforcement Learning - RL)

چارچوبی برای تصمیم‌گیری متوالی (Sequential Decision-Making) که در آن عامل با تعامل با محیط (مدل‌سازی شده به صورت فرآیند تصمیم‌گیری مارکوف - MDP)، پاداش تجمعی بلندمدت را حداکثر می‌کند.

هدف ریاضی: یادگیری سیاست \(\pi^*\) برای حداکثرسازی پاداش تنزیل‌شده تجمعی مورد انتظار:

\[ \text{Maximize } E \left[ \sum_{t=0}^{T} \gamma^t R_{t+1} \right] \]

که در آن \(\gamma \in [0, 1]\) عامل تنزیل (Discount Factor) است.

۱.۲. مدیریت داده و موازنه بایاس-واریانس (Bias-Variance Tradeoff)

تقسیم سه‌گانه داده (Train-Validation-Test)

اعتبارسنجی (Validation) برای تنظیم بیش‌مدل‌ها (Hyperparameters) و آزمون (Test) برای تخمین نهایی خطای تعمیم (Generalization Error) استفاده می‌شوند.

موازنه بایاس-واریانس

خطای کلی مورد انتظار یک مدل از سه جزء تشکیل شده است:

\[ \text{Error}_{\text{total}} = \text{Bias}^2 + \text{Variance} + \sigma^2 \]

بایاس بالا (کم‌برازش) و واریانس بالا (بیش‌برازش) دو چالش اصلی هستند.

بخش دوم: تحلیل عمیق رگرسیون خطی و گرادیان کاهشی

بررسی مدل هیپوتزیس، فرمولاسیون تابع هزینه MSE و جزئیات ریاضی الگوریتم بهینه‌سازی گرادیان کاهشی.

۲.۱. مدل هیپوتزیس و تابع هزینه MSE

مدل هیپوتزیس (Hypothesis Function)

مدل برای \(n\) ویژگی و بردار وزن \(\mathbf{w}\):

\[ h_{\mathbf{w}}(\mathbf{x}) = \mathbf{w}^T \mathbf{x} \]
تابع هزینه میانگین مربعات خطا (MSE)

این تابع محدب (Convex) برای کمینه‌سازی خطای مدل بر روی \(m\) نمونه آموزشی استفاده می‌شود:

\[ J(\mathbf{w}) = \frac{1}{2m} \sum_{i=1}^{m} (h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)})^2 \]

۲.۲. بهینه‌سازی توسط گرادیان کاهشی (Gradient Descent)

قانون به‌روزرسانی پارامترها

پارامتر \(w_j\) در جهت منفی گرادیان با نرخ \(\alpha\) به‌روزرسانی می‌شود:

\[ w_j := w_j - \alpha \frac{\partial}{\partial w_j} J(\mathbf{w}) \]
مشتق جزئی (گرادیان) تابع هزینه

فرمول نهایی گرادیان برای رگرسیون خطی:

\[ \frac{\partial}{\partial w_j} J(\mathbf{w}) = \frac{1}{m} \sum_{i=1}^{m} (h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)}) x_j^{(i)} \]
شبه‌کد ۱: الگوریتم گرادیان کاهشی دسته‌ای (Batch Gradient Descent) 

ALGORITHM Batch_Gradient_Descent (Data D, Learning_Rate α, Epochs T)
    Initialize weights W = {w_0, w_1, ..., w_n} randomly
    m = size of training set D
    
    FOR t FROM 1 TO T: // تکرار برای هر Epoch
        Gradient_Vector G = Initialize to zeros
        
        // محاسبه مجموع خطا روی کل داده‌ها
        FOR i FROM 1 TO m: 
            Error_i = Hypothesis(x^(i), W) - y^(i) 
            
            // محاسبه مشتق جزئی برای هر وزن w_j
            FOR j FROM 0 TO n: 
                G[j] = G[j] + Error_i * x_j^(i)
                
        // به‌روزرسانی همزمان تمام وزن‌ها:
        FOR j FROM 0 TO n:
            W[j] := W[j] - (α / m) * G[j]
            
    RETURN W

بخش سوم: تحلیل جامع رگرسیون لجستیک و تابع احتمال لگاریتمی

بررسی مدل احتمالاتی دسته‌بندی، تابع سیگموئید و تحلیل تابع هزینه Cross-Entropy.

۳.۱. مدل احتمالاتی و تابع سیگموئید (Logistic)

تابع سیگموئید (Sigmoid Function)

برای تبدیل خروجی خطی (\(z = \mathbf{w}^T \mathbf{x}\)) به یک احتمال \(\hat{P} \in [0, 1]\)، از تابع سیگموئید استفاده می‌شود:

\[ h_{\mathbf{w}}(\mathbf{x}) = \sigma(z) = \frac{1}{1 + e^{-z}} \]
احتمال مشروط (Conditional Probability)

تخمین احتمال تعلق به کلاس مثبت (\(Y=1\)) و منفی (\(Y=0\)) به صورت زیر است:

\[ P(y | \mathbf{x}; \mathbf{w}) = (h_{\mathbf{w}}(\mathbf{x}))^y (1 - h_{\mathbf{w}}(\mathbf{x}))^{1-y} \]

۳.۲. تابع زیان (Loss Function) Cross-Entropy

به جای MSE، در دسته‌بندی از تابع Log Loss یا Cross-Entropy استفاده می‌شود که جریمه سنگینی را برای پیش‌بینی‌های اشتباه با اطمینان بالا اعمال می‌کند.

تابع هزینه لگاریتمی (Log Loss/Cross-Entropy)

این تابع، منفی لگاریتم درست‌نمایی (Negative Log-Likelihood) داده‌های آموزشی است:

\[ J(\mathbf{w}) = - \frac{1}{m} \sum_{i=1}^{m} [y^{(i)} \log(h_{\mathbf{w}}(\mathbf{x}^{(i)})) + (1 - y^{(i)}) \log(1 - h_{\mathbf{w}}(\mathbf{x}^{(i)}))] \]
گرادیان کاهشی لجستیک

با وجود پیچیدگی تابع هزینه Cross-Entropy، فرم نهایی گرادیان آن شبیه به رگرسیون خطی است، که نشان‌دهنده یکپارچگی ریاضی این مدل‌ها است:

\[ \frac{\partial}{\partial w_j} J(\mathbf{w}) = \frac{1}{m} \sum_{i=1}^{m} (h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)}) x_j^{(i)} \]

بخش چهارم: رگولاریزاسیون (Regularization) و جلوگیری از بیش‌برازش

بررسی تکنیک‌های رگولاریزاسیون \(L_1\) و \(L_2\) برای کنترل پیچیدگی مدل و کاهش واریانس.

۴.۱. تعریف و هدف رگولاریزاسیون

رگولاریزاسیون یک مکانیزم برای جریمه کردن پیچیدگی مدل است. هدف آن افزودن یک جمله جریمه به تابع هزینه است تا وزن‌های مدل کوچک بمانند و مدل از داده‌های آموزشی فاصله نگیرد (کاهش واریانس).

تابع هزینه جدید (Regularized Cost Function)

تابع هزینه بهینه‌سازی شده، شامل تابع زیان اصلی و جمله رگولاریزاسیون است:

\[ J_{\text{reg}}(\mathbf{w}) = J(\mathbf{w}) + \lambda \cdot R(\mathbf{w}) \]

که در آن \(\lambda \ge 0\) عامل رگولاریزاسیون (Hyperparameter) است.

۴.۲. رگولاریزاسیون Ridge (\(L_2\)) و Lasso (\(L_1\))

۱. رگولاریزاسیون \(L_2\) (Ridge Regression)

جریمه بر اساس مجموع مربعات وزن‌ها اعمال می‌شود. این روش تمایل دارد وزن‌ها را به صفر نزدیک کند اما آن‌ها را دقیقاً صفر نمی‌کند.

\[ R(\mathbf{w}) = \sum_{j=1}^{n} w_j^2 \]

مزیت: جلوگیری از بیش‌برازش و تثبیت مدل.

۲. رگولاریزاسیون \(L_1\) (Lasso Regression)

جریمه بر اساس مجموع قدر مطلق وزن‌ها اعمال می‌شود. این روش می‌تواند برخی از وزن‌ها را دقیقاً صفر کند، که به انتخاب ویژگی (Feature Selection) کمک می‌کند.

\[ R(\mathbf{w}) = \sum_{j=1}^{n} |w_j| \]

مزیت: خودکارسازی انتخاب ویژگی و ایجاد مدل‌های ساده‌تر (Sparse Models).

بخش پنجم: ساختار درخت‌های تصمیم (Decision Trees) و معیارهای ناخالصی

تحلیل ساختار غیرپارامتریک درخت‌های تصمیم، معیارهای آنتروپی و شاخص جینی برای تقسیم گره‌ها.

۵.۱. ساختار درخت تصمیم و تقسیم تکراری

درخت تصمیم یک مدل غیرپارامتریک است که فضای ویژگی را با دنباله‌ای از تقسیم‌های ساده (Binary Splits) به نواحی مستطیلی تقسیم می‌کند. هر تقسیم بر اساس یک ویژگی خاص و یک آستانه (Threshold) صورت می‌گیرد.

۵.۲. معیارهای ناخالصی (Impurity Measures)

انتخاب بهترین تقسیم در هر گره، بر اساس بیشینه‌سازی کاهش ناخالصی (کسب اطلاعات) صورت می‌گیرد.

۱. آنتروپی (Entropy) و کسب اطلاعات (Information Gain)

آنتروپی، میزان عدم قطعیت یا تصادفی بودن اطلاعات در یک مجموعه داده \(S\) است:

\[ \text{Entropy}(S) = - \sum_{k=1}^{K} P_k \log_2(P_k) \]

کسب اطلاعات، میزان کاهش آنتروپی پس از تقسیم بر اساس ویژگی \(A\) را اندازه‌گیری می‌کند. الگوریتم ID3 بر اساس حداکثرسازی Gain عمل می‌کند:

\[ \text{Gain}(S, A) = \text{Entropy}(S) - \sum_{v \in \text{Values}(A)} \frac{|S_v|}{|S|} \text{Entropy}(S_v) \]
۲. شاخص جینی (Gini Index)

این معیار که توسط الگوریتم CART استفاده می‌شود، احتمال خطا در دسته‌بندی تصادفی را نشان می‌دهد. جینی نیز باید کمینه شود:

\[ \text{Gini}(S) = 1 - \sum_{k=1}^{K} P_k^2 \]
شبه‌کد ۲: الگوریتم ساخت درخت تصمیم (ID3/CART) 

ALGORITHM Decision_Tree_Train(Data D, Features F)
    // شرایط توقف (Stop Conditions)
    IF D is Pure (all samples same class) OR F is empty OR Max_Depth reached:
        RETURN Leaf_Node(Majority_Class(D))
        
    Best_Feature = NULL
    Max_Impurity_Reduction = -INF
    
    // جستجو برای بهترین تقسیم
    FOR EACH feature A in F:
        Calculate Impurity_Reduction(D, A) // (e.g., Information Gain)
        
        IF Impurity_Reduction(D, A) > Max_Impurity_Reduction:
            Max_Impurity_Reduction = Impurity_Reduction(D, A)
            Best_Feature = A
            
    // ایجاد گره جدید و تقسیم داده‌ها
    Tree_Node = Node(Best_Feature)
    FOR EACH value v of Best_Feature:
        D_v = Subset of D where A = v
        Tree_Node.Child[v] = Decision_Tree_Train(D_v, F - {A})
        
    RETURN Tree_Node

بخش ششم: ماتریس سردرگمی (Confusion Matrix) و معیارهای عملکرد

تحلیل جامع معیارهای ارزیابی در دسته‌بندی، با تمرکز بر Precision، Recall و F1-Score در شرایط عدم تعادل کلاس‌ها.

۶.۱. اجزای ماتریس سردرگمی

در یک دسته‌بندی دودویی (Binary Classification)، ماتریس سردرگمی، نتایج پیش‌بینی شده را در برابر واقعیت‌ها مقایسه می‌کند:

True Positives (TP) / True Negatives (TN)
پیش‌بینی‌های صحیح مدل.
False Positives (FP) - خطای نوع اول
مدل به اشتباه، کلاس منفی را مثبت پیش‌بینی کرده (مثبت کاذب).
False Negatives (FN) - خطای نوع دوم
مدل به اشتباه، کلاس مثبت را منفی پیش‌بینی کرده (منفی کاذب).

۶.۲. معیارهای مبتنی بر ماتریس برای داده‌های نامتعادل

در مسائل واقعی با عدم تعادل کلاس‌ها (Imbalanced Data)، Accuracy معیار کافی نیست و باید از معیارهای دقیق‌تری استفاده کرد:

۱. دقت (Precision)

نسبت موارد مثبت واقعی به کل مواردی که مدل به عنوان مثبت پیش‌بینی کرده است. (مربوط به هزینه FP). مدل چقدر محتاط است؟

\[ \text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}} \]
۲. فراخوان (Recall / Sensitivity)

نسبت موارد مثبت واقعی به کل موارد مثبتی که واقعاً در داده‌ها وجود داشته است. (مربوط به هزینه FN). مدل چقدر جامع است؟

\[ \text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}} \]
۳. امتیاز F1 (F1-Score)

میانگین هارمونیک Precision و Recall. این معیار برای ارزیابی عملکرد کلی در شرایط نامتعادل به شدت توصیه می‌شود:

\[ \text{F1-Score} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} \]
۴. صحت (Accuracy)

معیاری کلی که در داده‌های نامتعادل، ارزش کمی دارد.

\[ \text{Accuracy} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}} \]

بخش هفتم: شبیه‌سازی پایتون و مراجع آکادمیک

کدنویسی محاسبات معیارها و معرفی منابع اصلی مطالعاتی.

۷.۱. کد پایتون: شبیه‌سازی ماتریس سردرگمی و معیارها

این کد، نحوه محاسبه معیارهای کلیدی را با استفاده از کتابخانه استاندارد sklearn نشان می‌دهد:

شبیه‌سازی ماتریس سردرگمی و محاسبه معیارهای استاندارد 
from sklearn.metrics import confusion_matrix, precision_score, recall_score, f1_score
import numpy as np

# داده‌های واقعی (True Labels) و پیش‌بینی‌های مدل (Predicted Labels)
# فرض کنید 100 نمونه داریم: 20 مورد مثبت (1) و 80 مورد منفی (0) واقعی هستند.
y_true = np.array([0] * 80 + [1] * 20)
y_pred = np.array([0] * 78 + [1] * 2 + [0] * 5 + [1] * 15)
# نتیجه این پیش‌بینی: TP=15, TN=78, FP=2, FN=5 (جمعاً 100 نمونه)

# ۱. محاسبه ماتریس سردرگمی (خروجی: [[TN, FP], [FN, TP]])
conf_matrix = confusion_matrix(y_true, y_pred)
print("--- ماتریس سردرگمی ---")
print(conf_matrix)

# ۲. محاسبه معیارهای اصلی
precision = precision_score(y_true, y_pred)
recall = recall_score(y_true, y_pred)
f1 = f1_score(y_true, y_pred)
accuracy = np.mean(y_true == y_pred)

print("\n--- نتایج ارزیابی عملکرد ---")
print(f"Accuracy (صحت کلی): {accuracy:.4f}")
print(f"Precision (دقت): {precision:.4f}  (15 / 17)")
print(f"Recall (فراخوان): {recall:.4f}    (15 / 20)")
print(f"F1-Score: {f1:.4f}")
