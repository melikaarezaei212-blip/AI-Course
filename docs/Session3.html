جلسه ۳: روش‌شناسی حل مسئله و الگوریتم‌های جستجوی پیشرفته

تحلیل عمیق مدل‌های ریاضی جستجو، بهینه‌سازی منابع و الگوریتم‌های A* و IDA*

بررسی جزئیات Completeness، Optimality و پیچیدگی‌های محاسباتی الگوریتم‌های ناآگاهانه و آگاهانه.

بخش اول: مدل‌سازی حالت و فرمول‌بندی مسئله جستجو

۱.۱. تعریف فضای حالت و عناصر مسئله

یک مسئله جستجوی تک-عاملی (Single-Agent Search Problem) به صورت رسمی توسط چهار مؤلفه اصلی تعریف می‌شود. هدف اصلی، یافتن یک دنباله اعمال (مسیر) از وضعیت اولیه \(S_0\) به یکی از وضعیت‌های هدف \(\mathcal{G} \subset \mathcal{S}\) است که کمترین هزینه تجمعی را داشته باشد.

۱. وضعیت شروع (Initial State) \(S_0\)
نقطه‌ای که جستجو از آن آغاز می‌شود: \(S_0 \in \mathcal{S}\).
۲. مجموعه عملگرها و تابع جانشینی
مجموعه اعمال ممکن \(\mathcal{A}\) و تابع جانشینی (Transition Function) که نحوه تغییر وضعیت را مشخص می‌کند: \[ \text{SUCCESSOR}(s, a) \to s' \]
۳. آزمون هدف (Goal Test) و هزینه مسیر (Path Cost)
هزینه مسیر \(g(n)\)، هزینه تجمعی گام‌ها از \(S_0\) تا گره \(n\) است. هدف، کمینه‌سازی این مقدار است: \[ g(n) = \sum_{i=1}^{k} \text{Cost}(s_{i-1}, a_i, s_i) \]

۱.۲. معیار بهینه‌سازی منابع محاسباتی

الگوریتم‌های جستجو با چهار معیار اصلی ارزیابی می‌شوند. فرض می‌شود \(b\) عامل انشعاب، \(d\) عمق راه‌حل بهینه و \(m\) حداکثر عمق فضای حالت باشد.

معیار توضیح نماد ریاضی (مرتبه بزرگی)
Completeness (کامل بودن) تضمین یافتن راه‌حل در صورت وجود. بله/خیر
Optimality (بهینگی) تضمین یافتن کم‌هزینه‌ترین راه‌حل. \(\pi^* = \operatorname{argmin}_{\pi} g(\text{Goal})\)
Time Complexity (زمانی) تعداد گره‌های تولید شده. \(O(\cdot)\)
Space Complexity (حافظه) حجم حافظه برای ذخیره گره‌های در حال بررسی (Frontier). \(O(\cdot)\)

بخش دوم: تحلیل تفصیلی الگوریتم‌های جستجوی ناآگاهانه

الگوریتم‌های ناآگاهانه (Blind Search) هیچ اطلاعاتی از فاصله یا جهت هدف ندارند و در فضای حالت به صورت روشمند (Systematic) عمل می‌کنند.

۲.۱. جستجوی اول سطح (Breadth-First Search - BFS)

تئوری عملکرد
BFS یک الگوریتم کامل و بهینه است، به شرطی که هزینه هر گام (Edge Cost) برابر و متناهی باشد.
تحلیل پیچیدگی

تعداد گره‌های تولید شده: \(1 + b + b^2 + \dots + b^d = O(b^d)\)

  • Time Complexity: \(O(b^d)\).
  • Space Complexity: \(O(b^d)\). (به دلیل ذخیره تمام گره‌های لایه \(d\)).
نتیجه‌گیری علمی: BFS به دلیل پیچیدگی حافظه نمایی، در مسائل با عمق زیاد (مانند \(d>15\)) عملیاتی نیست.

۲.۲. جستجوی اول هزینه یکنواخت (Uniform Cost Search - UCS)

تئوری عملکرد
UCS نسخه بهینه‌سازی شده BFS برای گراف‌های با یال‌های وزن‌دار (هزینه متغیر) است. این الگوریتم گره‌ای را گسترش می‌دهد که کمترین \(g(n)\) را داشته باشد. \[\text{Selection} = \underset{n \in \text{Frontier}}{\operatorname{argmin}} \ g(n)\]
تحلیل پیچیدگی

پیچیدگی UCS نه به عمق \(d\)، بلکه به هزینه مسیر بهینه \(C^*\) و کمترین هزینه گام \(\epsilon > 0\) بستگی دارد. \(\epsilon\) تضمین می‌کند که تعداد گام‌ها نامتناهی نشود.

  • Time Complexity: \(O(b^{1+C^*/\epsilon})\).
  • Optimality: بله. UCS در هر شرایطی ارزان‌ترین مسیر را پیدا می‌کند.

بخش سوم: جستجوی آگاهانه (Informed Search) و مدل‌های اکتشافی

۳.۱. تابع ارزیابی \(f(n)\) در الگوریتم A* (A-Star Search)

الگوریتم A* با ترکیب هزینه واقعی تا گره \(n\) (\(g(n)\)) و هزینه تخمینی باقی‌مانده (\(h(n)\))، به صورت هوشمندانه مسیر را هدایت می‌کند.

\[ f(n) = g(n) + h(n) \]

A* به عنوان یک الگوریتم Best-First Search عمل می‌کند که تابع ارزیابی آن \(f(n)\) است.

۳.۲. شرایط بهینگی و مدل‌های ریاضی Heuristic

برای حفظ بهینگی (Optimality) A*، تابع اکتشافی \(h(n)\) باید دو شرط اساسی را برآورده کند:

۱. سازگاری (Admissibility)
تابع اکتشافی باید همواره یک کران پایین (Lower Bound) برای هزینه واقعی بهینه تا هدف (\(h^*(n)\)) ارائه دهد: \[ h(n) \le h^*(n) \quad \forall n \] تضمین: اگر \(h(n)\) سازگار باشد، A* بهینه است.
۲. یکنواختی (Consistency)
تخمین هزینه \(h(n)\) باید با هزینه واقعی گام‌ها سازگار باشد. برای هر یال از \(n\) به \(n'\) با هزینه \(c(n, n')\): \[ h(n) \le c(n, n') + h(n') \] تضمین: اگر \(h(n)\) یکنواخت باشد، A* در اولین باری که گره‌ای را گسترش می‌دهد، مسیر بهینه به آن گره را پیدا کرده است.

۳.۳. پیچیدگی و کارایی A*

پیچیدگی زمانی A* به کیفیت تابع اکتشافی بستگی دارد. اگر \(h(n)\) بسیار به \(h^*(n)\) نزدیک باشد، A* تقریباً به صورت خطی پیش می‌رود، اما در حالت کلی همچنان از مرتبه نمایی است: \(O(b^{d'})\) که \(d'\) معمولاً بسیار کوچکتر از \(d\) است. A* نیز مانند BFS، از نظر حافظه، یک الگوریتم ناکارآمد است (\(O(b^{d'})\)).

۳.۴. الگوریتم IDA* (Iterative Deepening A*)

برای غلبه بر مشکل حافظه A*، از IDA* استفاده می‌شود. این الگوریتم جستجوی DFS را با یک محدودیت آگاهانه (Informed Cutoff) بر اساس مقدار \(f(n)\) انجام می‌دهد.

فرآیند عملیاتی
IDA* به صورت تکراری با محدودیت \(L\) آغاز می‌شود. در هر تکرار، محدودیت \(L\) به کمترین مقدار \(f(n)\) که در تکرار قبلی از آن عبور شده، افزایش می‌یابد.
تحلیل کارایی
  • Optimality: بله (اگر \(h(n)\) سازگار باشد).
  • Space Complexity: \(O(b \cdot d)\). (بهترین در حافظه).
  • Time Complexity: \(O(b^d)\) (از نظر زمانی مشابه A* است، اما به دلیل تکرارها کمی کندتر).
نتیجه‌گیری: IDA* معمولاً الگوریتم بهینه انتخابی است زیرا تعادلی عالی بین بهینگی و مصرف کم حافظه ایجاد می‌کند.

بخش چهارم: شبیه‌سازی پایتون و مدل‌های اکتشافی پیشرفته

۴.۱. کد پایتون ۱: پیاده‌سازی اصلی A* با توضیحات علمی

پیاده‌سازی A* با توضیحات آکادمیک و مدیریت صف اولویت 
import heapq # استفاده از heapq برای پیاده‌سازی صف اولویت (Priority Queue)

# تابع Heuristic h(n) - تخمین Admissible فاصله تا هدف
def heuristic(node, goal='G'):
    # فرض بر این است که این مقادیر از یک تابع Consistent/Admissible استخراج شده‌اند.
    distances = {'S': 10, 'A': 8, 'B': 6, 'C': 4, 'D': 2, 'G': 0}
    return distances.get(node, float('inf'))

# تعریف گراف وزنی (مدل فضای حالت)
graph = {
    'S': [('A', 1), ('B', 5)], # یال‌ها: (گره مقصد, هزینه c(n, n'))
    'A': [('C', 2), ('D', 4)],
    'B': [('D', 1)],
    'C': [('G', 7)],
    'D': [('G', 3)],
    'G': []
}

def a_star_search(start_node, goal_node):
    # صف اولویت شامل: (f_cost, g_cost, node, path)
    priority_queue = [(0, 0, start_node, [start_node])]
    
    # دیکشنری g_costs برای نگهداری بهترین هزینه g(n) فعلی یافت شده برای هر گره
    g_costs = {start_node: 0} 

    while priority_queue:
        # استخراج گره با کمترین f(n) (تابع ارزیابی A*)
        f_cost, g_cost, current_node, path = heapq.heappop(priority_queue)

        if current_node == goal_node:
            # راه‌حل بهینه یافت شد (تضمین شده توسط Admissibility)
            return path, g_cost 

        # شرط پرش: گره‌ای که قبلاً با هزینه کمتر پردازش شده باشد نادیده گرفته می‌شود.
        if g_cost > g_costs.get(current_node, float('inf')):
            continue

        for neighbor, edge_cost in graph.get(current_node, []):
            # محاسبه هزینه واقعی مسیر جدید تا همسایه: g(n') = g(n) + c(n, n')
            new_g_cost = g_cost + edge_cost
            
            # اگر مسیر جدید به همسایه بهتر است (کم هزینه‌تر)
            if neighbor not in g_costs or new_g_cost < g_costs[neighbor]:
                g_costs[neighbor] = new_g_cost
                h_cost = heuristic(neighbor)
                
                # محاسبه تابع ارزیابی f(n') = g(n') + h(n')
                new_f_cost = new_g_cost + h_cost
                new_path = path + [neighbor]
                
                # اضافه کردن گره به صف اولویت
                heapq.heappush(priority_queue, (new_f_cost, new_g_cost, neighbor, new_path))
                
    return "Goal not reachable", float('inf')

# اجرای جستجو
optimal_path, final_cost = a_star_search('S', 'G')

print(f"مسیر بهینه (A*): {optimal_path}")
print(f"هزینه بهینه مسیر: {final_cost}")
مسیر بهینه (A*): ['S', 'A', 'D', 'G']
هزینه بهینه مسیر: 8

۴.۲. کد پایتون ۲: مدل اکتشافی Manhattan Distance

تابع Manhattan Distance (فاصله منهتن) یک Heuristic Admissible و Consistent برای مسائل حرکتی شبکه‌ای است، مانند پازل ۸ و مسیرهای شهری.

\[ h_{\text{Manhattan}}(n) = \sum_{\text{tiles}} \left( |x_n - x_{\text{goal}}| + |y_n - y_{\text{goal}}| \right) \]

محاسبه دقیق Manhattan Distance برای مسائل گسسته 
# مختصات وضعیت‌ها (برای یک شبکه فرضی)
node_coords = {
    'A': (1, 5), 'B': (3, 2), 'C': (5, 5), 'D': (7, 2), 'G': (8, 8) # G: وضعیت هدف
}

def manhattan_distance(node_name, goal_name='G'):
    """
    محاسبه فاصله منهتن. 
    این مقدار همیشه کران پایینی از تعداد گام‌های مورد نیاز در شبکه است،
    لذا یک Heuristic Admissible و Consistent است.
    """
    if node_name == goal_name:
        return 0
        
    x1, y1 = node_coords[node_name]
    x2, y2 = node_coords[goal_name]
    
    # محاسبه تفاضل قدر مطلق مختصات
    distance = abs(x1 - x2) + abs(y1 - y2)
    return distance

# تست تابع اکتشافی
nodes_to_test = ['A', 'B', 'C', 'D']
print("مقادیر Heuristic (Manhattan Distance) تا G:")
for node in nodes_to_test:
    h_value = manhattan_distance(node)
    print(f"  h({node}) = {h_value}")
مقادیر Heuristic (Manhattan Distance) تا G:
  h(A) = 10
  h(B) = 11
  h(C) = 6
  h(D) = 7
